# ZAI-CTS AI Architecture

AI is advisory, explainable, governed, and human-overridable. It never replaces regulatory authority.

## AI Modules

| Module | Purpose |
| --- | --- |
| PDD Copilot | Drafts and reviews project design documents against approved methodologies |
| Legal Copilot | Answers SI 48 of 2025 and Article 6 questions using approved sources |
| Additionality Engine | Assesses whether a project is additional to baseline activities |
| Leakage Detection | Flags leakage risks across nearby land-use changes |
| Fraud Detection | Detects suspicious MRV, trading, and evidence patterns |
| Price Forecasting | Forecasts price trends using marketplace and macro signals |
| Satellite Intelligence | Summarizes satellite observations and spatial risk |
| Executive Assistant | Produces regulator dashboards and briefing summaries |

## Governance Fields

Every AI output stores:

- `model_version`
- `prompt_version`
- `confidence_score`
- `explanation`
- `evidence_references`
- `human_override_status`
- `bias_monitoring_result`
- `drift_monitoring_result`
- `audit_event_id`
